Creates a client object used to do the following spawn new cmd command/process spawn new PowerShell Runspace Pool/Pipeline copy a file from localhost to the remote Windows host fetch a file from the remote Windows host to localhost.

This is just an easy to use layer on top of the objects WinRS and RunspacePool/PowerShell. It trades flexibility in favour of simplicity.

If your use case needs some of that flexibility you can use these functions as a reference implementation for your own functions.

Parameters
serverThe server/host to connect to
kwargsThe various WSMan args to control the transport mechanism, see pypsrp.wsman.WSMan for these args

Copies a single file from the current host to the remote Windows host.

This can be quite slow when it comes to large files due to the limitations of WinRM but it is designed to be as fast as it can be. During the copy process, the bytes will be stored in a temporary file before being copied.

When copying it will replace the file at dest if one already exists. It also will verify the checksum of the copied file is the same as the actual file locally before copying the file to the path at dest.

Parameters

destThe path to the destination file on the Windows host
configuration_nameThe PowerShell configuration endpoint to use when copying the file.

The absolute path of the file on the Windows host

Executes a PowerShell script in a PowerShell runspace pool.

This uses the PSRP layer and is designed to run a PowerShell script and not a raw executable.

Because this runs in a runspace, traditional concepts like stdout, stderr, rc's are no longer relevant. Instead there is a output, error/verbose/debug streams, and a boolean that indicates if the script execution came across an error. If you want the traditional stdout/stderr/rc, use execute_cmd instead.

Parameters
scriptThe PowerShell script to run
configuration_nameThe PowerShell configuration endpoint to use when executing the script.
environmentA dictionary containing environment keys and values to set on the executing script.
Returns
A tuple of
output A unicode string of the output stream
streams pypsrp.powershell.PSDataStreams containing the other PowerShell streams
had_errors bool that indicates whether the script had errors during execution

Will fetch a single file from the remote Windows host and create a local copy.

Like copy(), this can be slow when it comes to fetching large files due to the limitation of WinRM.

This method will first store the file in a temporary location before creating or replacing the file at dest if the checksum is correct.

Parameters

str pypsrp.client.Client.sanitise_clixml ( str clixml)
static

When running a powershell script in execute_cmd (WinRS), the stderr stream may contain some clixml.

This method will clear it up and replace it with the error string it would represent. This isn't done by default on execute_cmd for various reasons but people can call it manually here if they like.

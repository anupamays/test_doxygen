str urllib3.fields.format_header_param_rfc2231 ( str name,
_TYPE_FIELD_VALUE value )

Helper function to format and quote a single header parameter using the strategy defined in RFC 2231.

Particularly useful for header parameters which might contain non-ASCII values, like file names. This follows RFC 2388 Section 4.4 <https://tools.ietf.org/html/rfc2388#section-4.4>_.

Parameters
nameThe name of the parameter, a string expected to be ASCII only.
valueThe value of the parameter, provided as bytes or str`. :returns: An RFC-2231-formatted unicode string.

.. deprecated:: 2.0.0 Will be removed in urllib3 v2.1.0. This is not valid for multipart/form-data header parameters.

◆ format_multipart_header_param()

str urllib3.fields.format_multipart_header_param ( str name,
_TYPE_FIELD_VALUE value )

Format and quote a single multipart header parameter.

This follows the WHATWG HTML Standard_ as of 2021/06/10, matching the behavior of current browser and curl versions. Values are assumed to be UTF-8. The \\n, \\r, and "</tt> characters are percent encoded. .. _WHATWG HTML Standard: https://html.spec.whatwg.org/multipage/ @code form-control-infrastructure.html#multipart-form-data @endcode @param name @endcode The name of the parameter, an ASCII-only <tt>str</tt>. @param value The value of the parameter, a <tt>str</tt> or UTF-8 encoded <tt>bytes</tt>. :returns: A string <tt>name="value" with the escaped value.

.. versionchanged:: 2.0.0 Matches the WHATWG HTML Standard as of 2021/06/10. Control characters are no longer percent encoded.

.. versionchanged:: 2.0.0 Renamed from format_header_param_html5 and format_header_param. The old names will be removed in urllib3 v2.1.0.

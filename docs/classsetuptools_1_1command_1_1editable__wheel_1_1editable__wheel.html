Detailed Description

Build 'editable' wheel for development.

This command is private and reserved for internal use of setuptools, users should rely on setuptools.build_meta APIs.

Configure commands to behave in the following ways: 

   - Build commands can write to ``build_lib`` if they really want to...
     (but this folder is expected to be ignored and modules are expected to live
     in the project directory...)
   - Binary extensions should be built in-place (editable_mode = True)
   - Data/header/script files are not part of the "editable" specification
     so they are written directly to the unpacked_wheel directory.

◆ _run_build_subcommands()

setuptools.command.editable_wheel.editable_wheel._run_build_subcommands ( self)
protected
Issue #3501 indicates that some plugins/customizations might rely on:
  1. build_py not running
  2. build_py always copying files to build_lib

However both these assumptions may be false in editable_wheel. This method implements a temporary workaround to support the ecosystem while the implementations catch up.
